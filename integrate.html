One-time setup — a token to open the install pull request

  1. Create a fine-grained token ↗ — the required permissions are pre-filled.
  2. Repository access → Only select repositories → add your-org/your-repo.
  4. Generate, then paste it below.

Stored in this browser only (localStorage) and sent only to api.github.com — never to the page host or to CI, so it can never be used from another machine. It stays on this device so you can reuse it for more installs; Forget token (or revoking it on GitHub) removes it — do that on a shared computer.

One-time setup — a GitLab token to open the install merge request

  1. In GitLab, open User settings › Access tokens, then create a token with the api scope.
  2. Confirm Target repo above is the GitLab project path, for example group/subgroup/project.
  3. Paste the token below. The browser uses it to create a branch, commit the generated files, and open a merge request.

Stored in this browser only (localStorage) and sent only to the GitLab host below. For GitLab.com use https://gitlab.com; for a company GitLab server, paste that server URL.
