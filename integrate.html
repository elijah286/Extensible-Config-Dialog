Apply to a new repository

Add these LabVIEW CI capabilities to another repository. Pick what you want — get an install command and a ready-to-commit bundle.

← Back to dashboard

1. Your repository

Where to install. Required for the one-click install; the command-line option can also infer it from your git remote.

2. Capabilities

All available capabilities are selected by default — uncheck any you don't want. Dependencies are added automatically.

3. Install

Let the configurator open a pull request that adds these files to your repository — and, if you like, link it from your README and switch on GitHub Pages so the dashboard publishes itself. Nothing to download, unzip, or commit by hand. The first time, you paste a fine-grained token; the panel walks you through creating it with the permissions pre-filled. It stays in this browser so you don’t have to paste it again.

One-time setup — a token to open the install pull request

  1. Create a fine-grained token ↗ — the required permissions are pre-filled.
  2. Repository access → Only select repositories → add your-org/your-repo.
  4. Generate, then paste it below.

Stored in this browser only (localStorage) and sent only to api.github.com — never to the page host or to CI, so it can never be used from another machine. It stays on this device so you can reuse it for more installs; Forget token (or revoking it on GitHub) removes it — do that on a shared computer.
