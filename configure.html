Choose monitored dependency files on the Dependencies page. Monitoring controls dependency warnings and dependency-file change detection; installation before actions only happens when automatic dependency installation is enabled.

When activities run A “submission” is any change arriving at this repository. Standard git workflows map to three sources: a push to the default branch (a direct commit, or a merged pull request landing on the default branch), a pull request from a branch in this repository, and a pull request from a fork (an outside contributor). Tick which activities run for each source. Pull requests from forks run with a read-only token and no access to your repository secrets, so they can validate code but can't deploy or publish.

Decide what happens for each way a revision reaches this repository. For example, run Mass Compile and VIDiff on every pull request, but build the VI Snapshots gallery only when a commit lands on the default branch.
