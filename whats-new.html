Update your CI tooling

One click — Update now applies the latest tooling straight to your repository, with no pull request to open or merge.

What happens when you click Update now?
  1. Click Update now below. The first time, paste a one-time token — the dialog walks you through it (the same token the dashboard’s “Run now” uses).
  2. The Apply LabVIEW CI update workflow runs and commits the new version straight to your default branch — nothing to review or merge.
  3. That’s it. Your dashboard rebuilds and the badge turns green with the new version within a minute. If your default branch is protected (it requires pull requests), the update opens one instead and turns on auto-merge — it still finishes on its own wherever the repository allows it, otherwise just merge that pull request to finish.

One-time setup — a token to start the update

  1. Create a fine-grained token ↗ — opens with the name, resource owner, and Actions: Read and write already filled in.
  2. Repository access → Only select repositories → add . (This is the one step GitHub can’t pre-fill from a link.)
  3. Permissions → Repository permissions → Actions → Read and write. Required — selecting the repo alone gives a 403.
  4. Generate, then paste it below.

Stored only in this browser (localStorage); sent only to api.github.com — never to the repo, the page, or CI. The same token works for the dashboard’s “Run now”.

Finish the update — add a one-time token

The update is built, but the repository’s built-in token couldn’t apply it — either this release touches GitHub workflow files the built-in token can’t push, or it isn’t allowed to write to this branch. A one-time token below applies the update for you, finishing right here.

  1. Create a fine-grained token ↗ — opens with the name, owner, and Contents + Workflows + Pull requests + Secrets = Read and write already filled in.
  2. Repository access → Only select repositories → add . (The one step GitHub can’t pre-fill from a link.)
  3. Generate, then paste it below.

Encrypted in this browser (libsodium) and stored only as this repository’s TOOLING_UPDATE_TOKEN secret — sent only to api.github.com, never to this page or CI, and not kept in your browser.
